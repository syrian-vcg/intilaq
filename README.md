# الانطلاق (Intilaq)

لغة بيانات عربية بسيطة ومقياسية لتنظيم وتبادل البيانات بين الأنظمة المختلفة،
شبيهة بفكرة JSON لكن بكلمات محجوزة عربية وصياغة سهلة القراءة.

هوية المشروع: خلفية زرقاء (`#1565C0`) مع أيقونة مقص ✂️ (الكلمة المحجوزة "قص").

## مكوّنات المشروع

| المجلد | المحتوى |
|---|---|
| `android-app/` | تطبيق أندرويد مكتوب بالكامل بجافا: محرر نص يلوّن الصياغة + محرك تحليل/تنفيذ كامل (Lexer + Parser) يحوّل نص الانطلاق إلى بيانات و JSON |
| `vscode-extension/` | إضافة VS Code لتلوين الصياغة (Syntax Highlighting) لملفات `.intq` |

## قواعد صياغة اللغة

### كائن
```
كائن {
    اسم = "أحمد"
    خانة = 25
حاجز
}
```

### مجموعة (قائمة عناصر)
```
هوايات = مجموعة {
    حلقة { بند = "قراءة" }
    حلقة { بند = "برمجة" }
حاجز
}
```

### الكائنات المتداخلة
```
كائن {
    عنوان = كائن {
        مدينة = "الرياض"
    حاجز
    }
حاجز
}
```

### أنواع القيم المدعومة
- نص: `"..."`
- رقم: `25` أو `12.5`
- منطقي: `صحيح` / `خاطئ`
- فراغ: `فراغ`
- كائن متداخل: `كائن { ... حاجز }`
- مجموعة متداخلة: `مجموعة { ... حاجز }`

### تعليقات
أي سطر يبدأ بـ `#` يُعتبر تعليقًا ويُتجاهل عند التحليل.

### الكلمات المحجوزة (٢٩ كلمة)
```
اسم، معرف، مجموعة، حلقة، تعريف، حاجز، نص، كتلة، كائن، قص، لصق، نقل،
توريد، تصدير، تنزيل، طباعة، جمع، قسم، ترجمة، فقرة، خانة، صف، هنا،
اشعار، بريد، تسمية، حدث، نجح، خطاء
```
> ملاحظة: الكلمات المحجوزة التي لم تُستخدم في القواعد الأساسية حاليًا (مثل `تعريف`, `نقل`, `توريد`, `تصدير`, `تنزيل`, `طباعة`, `جمع`, `قسم`, `ترجمة`, `فقرة`, `صف`, `هنا`, `اشعار`, `بريد`, `حدث`) محجوزة لتوسعات مستقبلية للغة (أوامر تنفيذية)، ولا يمكن استخدامها كأسماء حقول عادية إلا داخل علامتي تنصيص.

## تطبيق الأندرويد — البنية البرمجية

كل الكود بجافا، بدون أي تبعية خارجية لمحرك اللغة نفسه:

```
com.intilaq.lang/         ← محرك اللغة (مستقل، قابل لإعادة الاستخدام في أي مشروع جافا/أندرويد)
    Keyword.java          enum للكلمات المحجوزة
    TokenType.java        أنواع الرموز المعجمية
    Token.java            تمثيل الرمز الواحد
    Lexer.java            المحلل المعجمي (نص → رموز)
    Parser.java           المحلل النحوي (رموز → بيانات)
    IntilaqValue.java     نموذج البيانات الموحّد (كائن/مجموعة/نص/رقم/منطقي/فراغ) + تحويل إلى JSON
    Intilaq.java          الواجهة الموحدة لاستخدام المحرك (parse, tryParse, toJson)
    IntilaqException.java استثناء مخصص لأخطاء التحليل مع رقم السطر والعمود

com.intilaq.app/          ← واجهة المستخدم
    MainActivity.java     الشاشة الرئيسية: محرر + أزرار + عرض النتيجة
    IntilaqEditText.java  EditText مخصص يلوّن الصياغة أثناء الكتابة (Syntax Highlighting حي)
```

### تشغيل المشروع
1. افتح مجلد `android-app/` في Android Studio (File → Open).
2. سيقوم Android Studio تلقائيًا بتحميل وإعداد Gradle Wrapper والتبعيات (Sync) — لا حاجة لأي إجراء يدوي.
3. شغّله على محاكي أو جهاز حقيقي (الحد الأدنى: Android 7.0 / API 24).

> ملاحظة: المشروع يحتوي على كل ملفات Gradle النصية (`build.gradle`, `settings.gradle`, `gradle.properties`) اللازمة، وAndroid Studio يكمل تلقائيًا أي ملفات ثنائية مفقودة في `gradle/wrapper/` عند أول فتح للمشروع.

### تشغيل اختبارات الوحدة
```
android-app/app/src/test/java/com/intilaq/lang/IntilaqTest.java
```
تحتوي اختبارات JUnit للكائنات، المجموعات، الكائنات المتداخلة، القيم المنطقية، التعليقات، حالات الأخطاء، وتحويل JSON.
شغّلها من Android Studio مباشرة (Run → اختبارات الوحدة) أو عبر:
```
./gradlew test
```
